# Product Requirements Document - HuoTong（货通）

**Author:** Hezhangcan
**Date:** 2026-03-09

## Executive Summary

HuoTong（货通） 是一款面向 4 人家庭式小型商户（厨具/日杂）的轻量级 H5/PWA 业务管理工具。产品以"极简定制"为核心策略，解决目标用户完全依赖纸质单据和人脑记忆进行经营管理所导致的时间浪费和信息遗漏问题。由家人兼开发者主导开发，确保产品完全贴合实际使用场景，所有操作追求"3秒内完成"。

### What Makes This Special

- **极致贴合**：开发者是家庭成员，零距离观察真实场景，每个操作流程量身定做
- **极致简单**：界面和操作复杂度对标微信，大字体、大按钮、不超过3步点击
- **数据联动**：以单据为纽带串联商品、客户、供应商、库存、账款五大模块，出一笔货自动触发全链路数据更新

## Project Classification

- **项目类型**：Web App（H5/PWA）
- **领域**：通用小型商户管理
- **复杂度**：低
- **项目背景**：Greenfield（全新项目）
- **技术形态**：移动端网页 + PC 端兼容，手机浏览器打开即用

## Success Criteria

### 用户成功

- 家人愿意主动使用系统而非被催着用
- 查价、开单、记账操作比翻纸质本子更快
- 日常经营中逐步不再依赖纸质单据
- 不需要女儿在场，其他家人也能独立完成日常操作

### 技术成功

- 数据不丢失、不出错
- 手机端页面加载时间 < 2 秒
- 支持多人同时使用不冲突
- 数据可备份恢复

### 可衡量结果

- 不漏账、不错价、省时间

## Product Scope

### MVP — 最小可行产品

7 个核心功能模块，以"单据"为数据纽带：

1. **数字出货/进货单** — 替代纸质单据，自动关联商品、客户、供应商
2. **商品价格库** — 模糊搜索查价，支持批量录入
3. **应收应付追踪** — 单据自动生成账款记录，按客户/供应商汇总
4. **客户信息管理** — 基本信息 + 历史交易关联
5. **供应商信息管理** — 基本信息 + 历史进货关联
6. **库存管理** — 出入货自动增减，实时查询
7. **智能识图录入** — 拍照识别纸质单据，自动提取商品、数量、价格、客户信息填充表单

### Growth Features（二期）

- 智能补货建议、畅销/滞销分析、商品目录展示
- 出货状态追踪、多价格体系、历史价格追踪、单据模板

### Vision（未来）

优先级由实际使用中暴露的需求决定，不提前过度规划。

## User Journeys

### 旅程一：父亲出货开单（核心日常场景）

**场景**：老客户张大哥打电话来要一批厨具，父亲接完电话需要开单、发货、记账。

**当前痛点**：父亲挂完电话，先从抽屉里翻出一沓空白单据，凭记忆写上商品名和价格，有些价格记不清要翻以前的进货单找。写完单据交给帮忙发货的家人，然后在本子上记下张大哥又欠了多少钱。整个过程容易写错价、忘记某些商品的库存已不足。

**使用 HuoTong（货通） 后**：
1. 父亲打开手机 → 点"新建出货单"
2. 搜索"张大哥" → 自动带出客户信息
3. 搜索商品关键字 → 选择商品、输入数量 → 价格自动带出
4. 系统提示"锅铲库存仅剩3件"→ 父亲知道该补货了
5. 确认出货单 → 库存自动减少、应收自动增加
6. 全程 < 2 分钟，零纸张

**啊哈时刻**：第一次在电话还没挂断时就完成了开单。

### 旅程二：母亲对账催款（周期性任务）

**场景**：每月底母亲需要整理所有客户的欠款情况，逐一催收。

**当前痛点**：翻遍几个月的纸质单据，用计算器一笔笔加，很容易漏掉某张单子或算错金额。经常跟客户发生"我没欠那么多"的纠纷。

**使用 HuoTong（货通） 后**：
1. 打开"应收账款" → 按客户查看汇总欠款
2. 点击某客户 → 看到每笔未付单据明细
3. 客户付款后 → 标记"已付"或"部分付"
4. 催款时有据可查，减少纠纷

**啊哈时刻**：第一次发现有笔3个月前的货款差点忘了催。

### 旅程三：女儿批量录入初始数据（冷启动）

**场景**：系统上线前，需要把现有商品信息、客户和供应商资料录入系统。

**流程**：
1. 女儿在电脑端打开 HuoTong（货通）
2. 从纸质记录和进货单中整理商品清单
3. 批量录入商品名称、规格、售价、进价
4. 录入常用客户（姓名、电话、地址）
5. 录入供应商信息（名称、联系人、主营品类）
6. 数据就绪，家人可以开始使用

**关键需求**：电脑端批量录入效率要高，支持表格式快速输入。

### 旅程四：父亲进货记账（进货场景）

**场景**：供应商送货到店，父亲需要核对并记录进货。

**使用 HuoTong（货通） 后**：
1. 新建进货单 → 选择供应商
2. 逐项添加到货商品和数量
3. 核对无误后确认 → 库存自动增加、应付自动记录
4. 下次供应商来结账时，查看应付明细直接对账

### Journey Requirements Summary

| 旅程 | 揭示的核心能力 |
|------|--------------|
| 出货开单 | 商品搜索、客户选择、单据创建、库存联动、应收联动 |
| 对账催款 | 应收汇总、按客户筛选、付款状态管理、历史单据查看 |
| 批量录入 | PC端适配、批量输入、数据导入 |
| 进货记账 | 供应商选择、进货单创建、库存联动、应付联动 |

## Web App Specific Requirements

### 技术架构概述

- **SPA 架构**：单页应用，减少页面切换等待
- **PWA 支持**：可添加到主屏幕，提供类原生体验
- **响应式设计**：手机端优先，PC 端兼容（女儿批量录入场景）
- **浏览器兼容**：微信内置浏览器 + 主流手机浏览器（Chrome、Safari）

### 关键技术决策

- **离线处理**：MVP 阶段不要求离线使用，需要网络连接
- **实时同步**：多人操作需要数据实时同步，避免冲突
- **SEO**：不需要，这是私有工具
- **无障碍**：大字体、高对比度、大触控区域（适合各年龄段，非标准 WCAG 要求）

### 实现考量

- 数据存储方案需支持多端同步和数据备份
- 前端框架选择应倾向学习曲线低、社区成熟的方案
- 部署方案应简单可靠，单人可维护

## Project Scoping & Phased Development

### MVP 策略

**MVP 理念**：问题解决型 MVP — 最小化功能集合，只要能让家人"用起来比不用好"即可。

**资源现实**：单人开发者，业余时间开发，需要控制 MVP 体量在可交付范围内。

### MVP Feature Set（Phase 1）

**核心用户旅程支持：** 出货开单、进货记账、对账催款、数据初始化

**必做能力：**
1. 商品管理（录入、查询、模糊搜索、价格查看）
2. 客户管理（录入、查询、查看历史）
3. 供应商管理（录入、查询、查看历史）
4. 出货单（创建、关联客户和商品、自动计算金额）
5. 进货单（创建、关联供应商和商品、自动计算金额）
6. 应收追踪（自动生成、状态标记、按客户汇总）
7. 应付追踪（自动生成、状态标记、按供应商汇总）
8. 库存管理（出入货自动增减、当前库存查询）
9. 智能识图录入（拍照识别纸质单据、自动填充表单、用户确认后提交）

### Post-MVP Features（Phase 2）

- 智能补货建议（库存低于阈值提醒）
- 畅销/滞销分析（销售数据统计）
- 商品目录展示（可分享给客户）
- 出货状态追踪（状态流转）
- 多价格体系（零售价/批发价/老客户价）
- 历史价格追踪（进货价变动记录）
- 单据模板（常用组合快速复制）

### Phase 3（Expansion）

根据实际使用反馈确定优先级。

### 风险缓解

- **技术风险**：选择成熟技术栈，避免使用过新的框架；MVP 功能少但要稳定
- **采纳风险**：前期双轨并行（纸质+数字），给家人适应时间；操作必须比纸笔更快
- **资源风险**：单人开发，MVP 控制在 8 个核心能力，避免功能膨胀

## Functional Requirements

### 商品管理

- FR1: 用户可以录入商品信息（名称、规格、售价、进价）
- FR2: 用户可以编辑和删除已有商品信息
- FR3: 用户可以通过关键字模糊搜索商品
- FR4: 用户可以查看商品列表并按名称或类别浏览
- FR5: 用户可以在电脑端批量录入商品数据

### 客户管理

- FR6: 用户可以录入客户信息（姓名、电话、地址）
- FR7: 用户可以编辑和删除已有客户信息
- FR8: 用户可以搜索和浏览客户列表
- FR9: 用户可以查看某客户的历史出货单和应收账款汇总

### 供应商管理

- FR10: 用户可以录入供应商信息（名称、联系人、电话、主营品类）
- FR11: 用户可以编辑和删除已有供应商信息
- FR12: 用户可以搜索和浏览供应商列表
- FR13: 用户可以查看某供应商的历史进货单和应付账款汇总

### 出货单管理

- FR14: 用户可以创建出货单并关联客户
- FR15: 用户可以在出货单中添加多个商品条目（商品、数量、单价）
- FR16: 系统可以自动计算出货单总金额
- FR17: 用户可以查看历史出货单列表并按日期、客户筛选
- FR18: 用户可以查看出货单详情
- FR19: 出货单确认后，系统自动扣减相关商品库存
- FR20: 出货单确认后，系统自动生成对应的应收记录

### 进货单管理

- FR21: 用户可以创建进货单并关联供应商
- FR22: 用户可以在进货单中添加多个商品条目（商品、数量、单价）
- FR23: 系统可以自动计算进货单总金额
- FR24: 用户可以查看历史进货单列表并按日期、供应商筛选
- FR25: 用户可以查看进货单详情
- FR26: 进货单确认后，系统自动增加相关商品库存
- FR27: 进货单确认后，系统自动生成对应的应付记录

### 应收应付管理

- FR28: 用户可以查看所有应收账款列表，按客户汇总
- FR29: 用户可以查看所有应付账款列表，按供应商汇总
- FR30: 用户可以将账款标记为"已付"或"部分付款"
- FR31: 用户可以查看某笔账款关联的原始单据
- FR32: 系统可以显示每个客户/供应商的总欠款金额

### 库存管理

- FR33: 用户可以查看当前所有商品的库存数量
- FR34: 系统在出货时自动扣减库存
- FR35: 系统在进货时自动增加库存
- FR36: 用户可以手动调整库存数量（盘点校正）
- FR37: 用户可以查看某商品的库存变动记录

### 智能识图录入

- FR42: 用户可以在新建出货单/进货单时，通过拍照或从相册选取纸质单据图片，系统调用大模型 Vision API 自动识别并提取商品名称、数量、单价、客户/供应商名称等信息，填充到单据表单中
- FR43: 识别结果填充表单后，用户可以检查、修改识别结果，确认无误后再提交

### 系统与数据

- FR38: 用户可以通过手机浏览器访问所有功能
- FR39: 用户可以通过电脑浏览器访问所有功能（尤其批量录入）
- FR40: 多个用户可以同时使用系统，数据实时同步
- FR41: 用户可以备份和恢复数据

## Non-Functional Requirements

### 性能

- 页面加载时间 < 2 秒（3G 网络环境）
- 搜索商品响应时间 < 500 毫秒
- 创建单据操作 < 3 秒完成确认
- 支持 4 人同时在线操作无卡顿

### 安全

- 数据传输使用 HTTPS 加密
- 用户访问需要登录认证（简单密码或邀请码即可）
- 数据存储不暴露在公网可直接访问

### 可用性

- 界面使用大字体（最小 16px）、大按钮（最小 44x44px 触控区域）
- 高对比度配色，适合各年龄段阅读
- 核心操作路径不超过 3 步点击
- 输入表单支持自动填充和智能建议
- 操作出错时提供清晰的中文错误提示
- 支持修改和撤销操作，容错设计

### 可靠性

- 数据不丢失：提交成功的数据永久保存
- 提供定期自动备份机制
- 系统异常时给出友好提示，不导致数据损坏
