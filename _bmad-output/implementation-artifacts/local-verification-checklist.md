# 本地验证清单

所有循环共用此文件。每轮 DS 完成后，agent 会在此追加**该次循环**对应 story 的本地验证项；完成验证后请将对应条目的 **已验证** 改为 `是`。

**条目顺序**：按 story 编号顺序排列（3-2 → 3-3 → 3-4 → 4-1 → 4-2 → 4-3 …）；新增时插入到正确位置。

---

## 2-4-batch-import-products（循环完成日：2026-03-10）

**已验证**：否

1. 登录后使用**电脑浏览器**（或浏览器窗口宽度 ≥768px）访问商品列表 `/products`，应看到「批量录入」按钮（与「新增商品」按钮一起）。
2. 点击「批量录入」或直接访问 `/products/batch`，应进入批量录入页，显示表格（表头：名称、规格、售价、进价），至少一行可编辑，有「添加一行」「批量保存」按钮。
3. 在表格中填写多行商品（名称必填，规格/售价/进价可空），点击「批量保存」，应 Toast「成功录入 N 件商品」并跳转回商品列表，列表中能看到新录入的商品。
4. 再次进入批量录入页，任一行留空「名称」点击「批量保存」，该行应高亮（如红色背景），Toast 提示「第 X 行：请填写商品名称」，且不发起保存请求（列表无新增）。
5. 使用**手机浏览器**或将窗口缩小至窄屏（<768px）访问 `/products/batch`，应自动跳转到 `/products/new`（新增商品页）并 Toast「批量录入请使用电脑端」；在窄屏下商品列表页不显示「批量录入」按钮。

## 3-1-customer-entry-list（循环完成日：2026-03-10）

**已验证**：否

1. 未登录时访问 `/customers`，应被重定向到 `/login?redirect=/customers`。
2. 登录后直接访问 `/customers`，应进入客户列表页，顶部标题为「客户」；无数据时显示「暂无客户，点击下方按钮新增」及「新增客户」按钮。
3. 点击「新增客户」或访问 `/customers/new`，应显示表单：客户姓名（必填）、电话、地址；姓名为空点击「保存」应 Toast「请填写客户姓名」。
4. 填写姓名后保存，应 Toast「保存成功」并跳回客户列表，新客户出现在列表中（显示姓名、电话）。
5. 在列表页搜索框输入姓名或电话关键字（防抖约 300ms），列表应过滤为包含该关键字的客户；清空搜索应恢复全部列表。
6. 列表页下拉刷新，应重新拉取数据；无匹配时显示「没有找到相关客户」。

## 4-1-sale-order-create（循环完成日：2026-03-10）

## 4-4-photo-recognize-sale（循环完成日：2026-03-10）

**已验证**：否

1. **Edge Function 密钥**：在 Supabase Dashboard → Edge Functions → recognize-receipt → Secrets 中设置 `OPENAI_API_KEY`（OpenAI API Key），否则拍照识别会返回 500。
2. 登录后进入新建出货单页 `/sale-orders/new`，应看到「拍照识别」「添加商品」「保存」三个按钮。
3. 点击「拍照识别」→ 选择一张纸质出货单图片（或手机拍照）→ 应显示「正在识别...」遮罩；若 Edge Function 正常且图片清晰，识别成功后自动填充客户（匹配 customers 表）与商品条目（匹配 products 表）；未匹配到客户时 Toast「未匹配到客户「XXX」，请手动选择」；未匹配到的商品以「需手动选择：商品名」占位显示，可点击该行的「选择商品」按钮替换为实际商品。
4. 识别失败（模糊图、非单据图、或 API 错误）时，应 Toast「识别失败，请重新拍照或手动录入」，不覆盖当前已填表单。
5. 存在「需手动选择」占位行时点击「保存」，应 Toast「请为「需手动选择」的条目选择商品」；为所有占位行选择商品后，可正常保存草稿并继续确认出货流程。
6. （可选）在 Supabase Dashboard 中确认 Edge Function `recognize-receipt` 已部署且状态为 ACTIVE；本地可用 `supabase functions serve recognize-receipt` 配合 `.env` 中 OPENAI_API_KEY 做联调。

## 6-3-payment-status（循环完成日：2026-03-10）

**已验证**：否

1. 登录后进入「查应收」或 `/receivables`，若有未付/部分付的应收记录，展开某客户后每条明细应显示「标记付款」按钮；已付（paid）记录不显示该按钮。
2. 点击某条应收的「标记付款」，应弹出底部面板，显示应收总额、已付金额、剩余未付金额，以及「本次付款金额」输入框和确认/取消按钮。
3. 输入本次付款金额等于剩余未付并确认，应 Toast「已更新」，该条状态变为「已付」，已付金额更新，列表与汇总刷新；输入金额小于剩余未付并确认，状态变为「部分付款」，已付金额累加。
4. 输入本次付款金额大于剩余未付并点击确认，应 Toast「付款金额不能超过未付金额」，不更新数据，弹窗保持打开。
5. 进入「查应付」或 `/payables`，对未付/部分付的应付记录执行上述 2–4 的同样操作，行为一致（应付总额、已付、剩余未付、标记付款后刷新）。

## 6-4-receivable-payable-detail（循环完成日：2026-03-10）

**已验证**：否

1. **应收 → 查看原始单据**：登录后进入「查应收」`/receivables`，展开某客户，在任一条应收明细上点击「查看原始单据」，应跳转到该笔应收关联的出货单详情页（`/sale-orders/:id`），详情内容与应收记录一致。
2. **应付 → 查看原始单据**：进入「查应付」`/payables`，展开某供应商，在任一条应付明细上点击「查看原始单据」，应跳转到该笔应付关联的进货单详情页（`/purchase-orders/:id`），详情内容与应付记录一致。
3. **客户详情页**：进入客户列表 `/customers`，点击某客户进入详情页（`/customers/:id`），页面底部应显示「应收账款汇总」区块（总欠款：¥xxx，若无欠款则显示 ¥0.00）和「历史出货单」区块；历史出货单列表显示该客户的出货单（单号、日期、金额、状态），最多 20 条；点击某条出货单应跳转到该出货单详情页。
4. **供应商详情页**：进入供应商列表 `/suppliers`，点击某供应商进入详情页（`/suppliers/:id`），页面底部应显示「应付账款汇总」区块（总欠款：¥xxx）和「历史进货单」区块；历史进货单列表显示该供应商的进货单（单号、日期、金额、状态），最多 20 条；点击某条进货单应跳转到该进货单详情页。
