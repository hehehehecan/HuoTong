# Android 真机回归与反馈闭环（Story 12.2）

本文档用于家庭内测阶段，目标是在每次候选包发布前后，形成可执行的真机回归流程和统一反馈格式，帮助开发者快速判断是否需要重新发版。

## 1. 回归前准备

执行回归前，先确认：

1. 候选 APK 已完成签名并可正常安装。
2. 目标设备至少包含 1 台家庭真实使用手机（非模拟器）。
3. 已记录本次候选版本号（`versionName` / `versionCode`）。
4. 网络环境可切换（Wi-Fi / 4G / 5G），便于做弱网验证。

## 2. 核心流程回归清单（至少 1 台真机）

每一项均记录「通过 / 失败 / 备注」：

1. 登录（账号密码正确可进入首页，错误提示清晰）。
2. 商品模块（列表加载、搜索、新增/编辑基础流程可用）。
3. 客户模块（列表、搜索、详情可访问）。
4. 供应商模块（列表、搜索、详情可访问）。
5. 出货流程（创建草稿、确认关键交互、详情可查看）。
6. 进货流程（创建草稿、确认关键交互、详情可查看）。
7. 应收应付（列表可加载、状态更新流程可执行）。
8. 库存模块（库存总览与变动记录可访问）。
9. 前后台切换（切到后台再恢复，页面与会话状态可恢复）。

## 3. 弱网与异常验证

1. 切换弱网/断网后，关键页面出现中文可理解提示。
2. 网络恢复后可通过重试继续操作，不需要强制重装 App。
3. 对于失败场景，记录触发操作、错误提示和恢复方式。

## 4. 反馈采集模板（家人提交）

反馈问题时，至少提供：

1. 版本号
2. 手机机型
3. 网络环境（Wi-Fi / 4G / 5G）
4. 问题复现步骤（建议 1、2、3 步）
5. 问题现象描述（可附截图/录屏）

可直接复制以下模板：

```text
【货通问题反馈】
- 版本号：
- 手机机型：
- 网络环境（Wi-Fi / 4G / 5G）：
- 问题复现步骤：
  1.
  2.
  3.
- 问题现象描述：
- 是否可通过重试恢复（是/否）：
```

## 5. 维护者快速分级与发版判断

收到反馈后，优先判断：

1. 是否阻断核心流程（登录、开单、库存、账款）。
2. 是否稳定复现（每次必现 / 偶发）。
3. 是否可通过切网、重试或重启临时恢复。
4. 影响范围是否仅个别机型或网络。

建议决策：

- **需要重新发版**：阻断核心流程，或无法通过临时手段恢复。
- **可先观察**：不阻断核心流程且可稳定规避，先记录并纳入下个版本修复。
